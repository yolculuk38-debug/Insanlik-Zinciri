# Security Policy

## Purpose

Humanity Chain is focused on transparent records, verification workflows, and archive integrity.

This security policy defines how risks, misuse, and integrity concerns should be handled.

---

## Security Principles

- Records should be traceable.
- Archive changes should be documented.
- Verification references should remain transparent.
- Hash and QR references should not be used to mislead users.
- Sensitive personal information should not be archived without clear consent.

---

## Integrity Risks

Potential risks include:

- Fake records
- Manipulated screenshots
- Misleading AI attribution
- Broken verification links
- QR phishing
- Hash mismatch
- Unauthorized modification claims

---

## Reporting Issues

Security or integrity concerns should be reported through GitHub Issues.

When reporting a concern, include:

- Affected file or record
- Description of the issue
- Evidence or context
- Suggested correction, if available

---

## Human Oversight

Human review remains required for sensitive archive decisions.

AI-generated analysis may support review, but it does not replace human responsibility.

---

## Experimental Status

Humanity Chain is an experimental open verification protocol.

Security processes may evolve as the project grows.
